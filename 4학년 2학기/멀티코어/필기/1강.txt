#컴퓨터
* 직렬 컴퓨터(Serial Computer)

* 병렬 컴퓨터(Parallel Computer)
여러개의 CPU를 갖는 컴퓨터
직렬 컴퓨터의 속도 제한 극복을 위해 제작
현재 사용하고 있는 컴퓨터
>>하지만... 하드웨어 설계가 병렬일 뿐이지 병렬을 써먹기 위해서는 직접 멀티쓰레딩을 해줘야 한다.

#병렬 컴퓨터
컴퓨터가 여러개 있는게 아니다 (-> 이건 필요가 없다)
있는 이유? 하나의 작업을 보다 빨리 실행하기 위해서.
->작업을 여러개 나눠서 한다.

코어가 많다고 속도 증가가 있는것은 절대 아니다... 0%!
왜냐하면? 프로그램이 직렬이기 때문이다.

*병렬 프로그램의 특징
여러 곳이 동시에 실행된다.

코어 하나마다 프로그램 카운터가 하나씩 존재한다.
-> 프로그램을 나눌수 있는 수

쓰레드 끼리 정보공유, 등 잘 맞춰서 동작되도록 -> 동기화 작업

여러 모델이 존재한다. 하지만 수업시간에는 공유메모리 모델을 다룰것
			(Shared Memory)
메시지 패싱도 있는데 별로 안쓰인다.

*병렬 프로그램의 요구사항 : 정확성과 성능
-> 왜 멀티쓰레드 프로그램 함? -> CPU가 졸라 느리니까

프로그램이 느리다. -> 성능개선(자료구조, 알고리즘) -> 느린원인?
1-> I/O -> I/O 최적화
2-> CPU -> 멀티쓰레드 프로그래밍
